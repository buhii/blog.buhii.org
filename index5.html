2015-11-24

Gender symbols

Twitter を徘徊していたとき NY 在住の女性ソフトウェアエンジニアのプロフィールに "⚢" とあった。Unicode に10種類以上の gender symbols があるというのは知らなかった。Wikipedia によると2005年から取り込まれているようだ。

学生の頃、同性から好きだと告白されたことがある。彼は僕と同じ寮にいて、ときどき彼と寮のお風呂で出会うこともあった。彼とは恋人にはならなかったが、同じ体育の授業を取ったり都内に住む彼の親戚の家へ遊びにいったり、引き続き友達として付き合っていた。

今年の夏にフィンランドに旅行した時、ヘルシンキから１駅ほど離れた公園で "mini-Pride" なるイベントに出くわした。案内してくれた友達のフィンランド人女性いわく、Pride というイベントがヘルシンキで開催されるついでに、この小さな公園でも小規模なイベントが開催されているのではないか、ということだった。公園内には老若男女たくさんの人たちが詰めかけていた。そこでは大音量で BGM が流れる中、シャンパンを飲みながら男性がかなり派手な女装をして公園内をマラソンしていた。公園内の人たちは男女問わず彼らを応援し、彼らのユーモラスな振る舞いに歓声を送っていた。出演者の男性達は、公園で観戦していた子供たちに対して優しく接していた。このようなイベントに子供たちを連れてくるのは意外だったが、異性愛に限らず様々な愛があるということを小さいころから知るのは良いことなのかもしれない。イベントが終わった後、自分たちの前にいた若い女性二人組が静かにキスしながら公園を後にしていたのが印象的だった。

ヘルシンキ現代美術館の Kiasma に行った時もちょうど Robert Mapplethorpe や Tom of Finland が展示されていた。Tom of Finland がフィンランドの郵便局から切手として販売されているのはさすがにびっくりだけど。その数日後ストックホルムへ旅行した時も、虹色のポスターを街角の様々なところで見かけた。

案内してくれた友達から、アメリカ国籍を持っているけどフィンランドに帰化したいという男性の話を聞いた。フィンランドは2002年から同性同士の結婚 (registered partnerships) が認められており、フィンランド人の彼氏と結婚したいからというのが理由だったと思う。彼女はフィンランドでは男性に対して半年〜１年の兵役か社会奉仕が義務付けられているが、確か30歳を過ぎて帰化した場合その義務はなくなるのでそれまで待った方が良い、とアドバイスしたようだ。

日本でも渋谷区のように registered partnerships は徐々に広まっていくと思うが、カミングアウトできる人はそう簡単には増えないような気がする。いずれにしろ同性同士、異性同士問わず隠さなくても好きな人と一緒になれる社会になれば良いね。

messnamed

Max6 では（環境の制約で Max7 が使えないのだ）ヘビーに js object を使っているのだけど、最近まで messnamed という関数を知らなかった。今までは以下のように js object の outlet からパッチコードを引いて使っていた。

outlet(0, ["hello", "world", 1]);

しかしこの方法だと他にもいろいろなところに情報を送ろうとするたびに js object の outlet を増やさないとならず outlet の管理やパッチコードのメンテナンスが大変になる。

ひとつの方法としては、js object の下に route object を配置してそこで分岐させる方法が考えられる。

どの outlet がどこに対応しているかを js object で管理する必要がなくなるので多少マシになる。しかしパッチコードの複雑さは解決していない。そこで messnamed の出番だ。

messnamed("target1", ["Hello", "world", 1]);
messnamed("target2", ["hoge", "fuga", 2]);
messnamed("target3", ["foo", "bar", "bazz"]);
...

と書いておき、 r targetObject と receive object を作っておけばパッチコードを繋げることなく直接 js から情報を送ることができる。

さてこれで問題は解決した。欠点としては receive object がどこから接続されているかという情報が欠落してしまうことだ。receive object をダブルクリックすればどこから send されているか分かるが js 内からの情報はさすがにトレースされない。

どの情報をパッチコード経由で送るか、js 内部に隠蔽して送るかはちょっと考える必要がある。パッチコードで送りづらいものは js 内部で送ればいいと思うけど、Max ユーザの人たちはどのように使い分けているのだろう？
