2016-01-09

ごちうさ

ごちうさ難民になってしまった。

ご注文はうさぎですか？(？)の素晴らしさは、もはやここで語る必要すら無いだろう。毎週の放送時間がとても楽しみだった。 ローソンでコラボのお菓子を買ったり、チノちゃんのバースデーパーティーに参加してみんなでチノちゃんと水瀬いのりさんのお誕生日のお祝いをした。 ごちうさ1期11話のクリスマス話を観なければ、そして2期の最終話に絶対に間に合わせるんだ！という意気込みが無ければ、去年のクリスマス案件を頑張ることはできなかったに違いない。

なぜこれほど、ごちうさにハマったのか自分でも良くわからない。一つは瞳のハイライトの描き方があるような気がする。個人的にふわっとしたテイストのイラストが好きなのだけど、これまで自分が観てきたアニメは、瞳の部分がベタ塗りになっているものしか知らなかった。ごちうさは瞳のハイライトの極端な輝度勾配がそこまで無いために、違和感なく楽しめたのではないかと思う。最近はこういう表現も増えているのかもしれない。

日付が変わって、本日12月2日はチノ役の水瀬いのりさんのお誕生日です！ 水瀬さん、お誕生日おめでとうございます！！ #gochiusa #水瀬いのり生誕祭 pic.twitter.com/j9I4AvquNO

— TVアニメ『ご注文はうさぎですか？？』 (@usagi_anime) December 1, 2015

他にもTVアニメ公式ガイドブックを見て驚いたのが、背景美術が緻密に設計されていることだ。ボトルシップやクリスマスマーケット等は瞬間的なコマではもったいないと感じるくらいだ。

キャラのかわいさにこだわって、視聴者の方々に対して、作品を観ている最中に変な違和感を持たせないようにしたかった
(ご注文はうさぎですか？ TVアニメ公式ガイドブック Memorial Blend, p.124)

とあるのも頷ける。とにかく様々な要素が徹底的に作りこまれているのが印象的だった。最終話まで集中力を持続させて皆で力をあわせて作品を作り上げるのは、大変な労力だったに違いない。

たくさんの音楽CDが発表されているのも良かった。どれも素敵な曲ばかりで、プログラムを書くときも、コインランドリーに洗濯物を出しに行くときも、ごちうさの曲を何百回と聴いている。いつもと違うキャラのカップリングで曲がいくつか出ているのも楽しい。

昔からコーヒーが好きで手動のミルで豆を挽いて飲んでいたけれど、ごちうさを観てからもっとコーヒーの世界を探求したいと思うようになった。家の近くの素敵な焙煎所を見つけたり、コーヒーをもっと楽しめるようになったのは、チノちゃんのおかげだ。

2期が終わって悲しいけれど、Koi先生をはじめ、制作スタッフの方々にはゆっくり休んで充電してもらい、また素晴らしい作品をぜひ作っていただきたい。チノちゃんにはかなわないけど、美味しいコーヒーを淹れる練習をしつつ、のんびりお待ちしております。

2015-12-13

Gerhard Richter "Painting"

Gerhard Richter "Painting" に行ってきた。六本木WAKO WORKS OF ARTで2015年12月19日まで。ギャラリーは思ったよりも小さく、小部屋にいくつか分かれていた。絵具の物理的な性質をうまく利用して作品を作り上げているという印象。自然に任せられるところは任せるが、全体はコントロールするという感じ。細部までしっかり作りこまれていると感じた。かなり好み。写真の上に絵具を塗りつけた作品は、面白さがあまりわからなかった。本人が写真論／絵画論を語った本が展示されていたので、そのうち読んでみたい。会社の人からドイツのドレスデンにゲルハルト・リヒターの作品をたくさん展示している美術館があってすごく良かったと教えていただいたので、ドイツに行くときは観に行こう。

ゲルハルト・リヒターの作品でぜひ一度観てみたいのは、ケルン大聖堂 (Cologne Cathedral) のステンドグラス作品 だ。コンピュータによって計算されたランダムに配置された72色のステンドグラスが並べられているらしい。中学生の頃 Visual Basic 6 で今風で言う algorithmic design? なスクリーンセーバを作ったとき、random を使ってピクセルの色と位置を決定しているにも関わらず対称的な図が得られてびっくりしたことがある。今思えばヘボい擬似乱数を使っていたからそうなったのだと思うけど、それはそれで美的価値があるのかもしれない。最近 HN で話題になってた V8 JavaScript engine の擬似乱数問題も興味深い (TIFU by using Math.random())。真の乱数は Hardware random number generator で作るみたいだが、これは radiation source を使ったりするらしい。でもどうやって uniform な乱数に変換しているんだろう？下手に作ったらバイアスが入ったりしそうで怖い。設計方針とかあればぜひ読んでみたい。
